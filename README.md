# Privacy-Preserving NER LLM Proxy

## Кратко
Конвейер подготовки русского текста перед передачей в LLM, который обнаруживает и маскирует чувствительные сущности. Система объединяет NER и правила по шаблонам, чтобы повысить полноту обнаружения и устойчивость на шумных запросах.

## Задача
Перед передачей пользовательских запросов в LLM важно удалить или скрыть персональные и чувствительные данные. Одной модели NER для этого часто недостаточно: часть сущностей выражается нестандартно, с ошибками или в виде шаблонов. Поэтому в проекте объединены статистическое распознавание сущностей и правила по шаблонам.

## Что улучшено
- текст нормализуется до поиска сущностей;
- обнаружение выполняется двумя независимыми механизмами: NER и правилами по шаблонам;
- результаты объединяются перед маскированием;
- повышена устойчивость на запросах с шумом, смешанными форматами и нестрогими формулировками.

## Архитектура
```text
[входной текст]
      |
      v
[нормализация]
      |
      +-------------------+
      |                   |
      v                   v
    [NER]      [правила по шаблонам]
      |                   |
      +---------+---------+
                |
                v
 [объединение найденных сущностей]
                |
                v
         [маскирование]
                |
                v
 [текст, подготовленный для LLM]
```

## Структура репозитория
```text
notebooks/  исследовательские материалы, если используются
src/        основная логика обработки
```

## Критерии качества
- полнота обнаружения чувствительных сущностей;
- точность маскирования;
- устойчивость на шумных и смешанных форматах записи;
- корректность обработки русского текста.

## Ключевой вывод
Объединение NER и правил по шаблонам даёт более надёжную защиту входных данных, чем использование только одного механизма. Основной практический эффект — снижение числа пропущенных чувствительных сущностей перед обращением к LLM.

## Ограничения
- набор сущностей и правил нужно расширять под конкретную предметную область;
- часть редких сущностей требует отдельной донастройки;
- при изменении форматов ввода правила по шаблонам нужно поддерживать и пересматривать.
