# Privacy-Preserving NER for LLM Requests

Конвейер предобработки русскоязычных запросов перед передачей в LLM. Решение извлекает чувствительные сущности, нормализует их и маскирует, чтобы снизить риск утечки персональных данных.

## Назначение

Автоматически находить в пользовательском запросе чувствительные сущности и заменять их безопасными маркерами без потери основного смысла запроса.

## Архитектура

```text
Входной запрос
      │
      ▼
Извлечение сущностей
      │
      ▼
Нормализация
      │
      ▼
Проверка регулярными правилами
      │
      ▼
Маскирование
      │
      ▼
Безопасный запрос для LLM
```

## Что реализовано

- извлечение чувствительных сущностей из русскоязычного текста;
- нормализация найденных значений;
- дополнительная проверка регулярными правилами;
- маскирование сущностей безопасными маркерами;
- подготовка запроса к передаче в LLM.

## Какие типы сущностей обрабатываются

- персональные данные;
- адреса;
- телефоны;
- электронная почта;
- числовые идентификаторы и иные чувствительные поля.

## Структура репозитория

- ноутбуки и скрипты для извлечения сущностей;
- правила нормализации и маскирования;
- вспомогательные материалы для проверки качества.

## Оценка качества

Основные метрики:

- точность извлечения сущностей;
- полнота извлечения сущностей;
- доля корректно замаскированных сущностей;
- доля запросов, в которых после маскирования не остаётся исходных чувствительных фрагментов.

## Результаты

Численные значения не приводятся: в репозитории нет сохранённого отчёта с формальной оценкой после стандартного прогона. При появлении размеченной выборки и сценария оценки показатели фиксируют в отдельном отчёте и кратко резюмируют здесь.

## Ограничения

- качество зависит от выбранного способа извлечения сущностей;
- редкие и нестандартные форматы чувствительных данных требуют расширения правил;
- для воспроизводимого сравнения сохраняют результаты отдельно для варианта только с извлечением сущностей и для гибридного варианта с правилами.
