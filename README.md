# Privacy-Preserving NER for LLM Requests

Конвейер предобработки русскоязычных запросов перед передачей в LLM. Решение извлекает чувствительные сущности, нормализует их и маскирует, чтобы снизить риск утечки персональных данных.

## Задача
Нужно автоматически находить в пользовательском запросе чувствительные сущности и заменять их безопасными маркерами без потери основного смысла запроса.

## Архитектура решения
```text
Входной запрос
      │
      ▼
Извлечение сущностей
      │
      ▼
Нормализация
      │
      ▼
Проверка регулярными правилами
      │
      ▼
Маскирование
      │
      ▼
Безопасный запрос для LLM
```

## Структура репозитория
- ноутбуки и скрипты для извлечения сущностей;
- правила нормализации и маскирования;
- вспомогательные материалы для проверки качества.

## Какие сущности обрабатываются
- персональные данные;
- адреса;
- телефоны;
- электронная почта;
- числовые идентификаторы и другие чувствительные поля.

## Метрики
- точность извлечения сущностей;
- полнота извлечения сущностей;
- доля корректно замаскированных сущностей;
- доля запросов, в которых после маскирования не остаётся исходных чувствительных фрагментов.

## Результаты
В финальной версии README нужно указать:
- точность и полноту извлечения;
- долю корректного маскирования;
- сравнение между вариантом только с NER и вариантом с дополнительной проверкой регулярными правилами.

Формат записи:
- `Добавление регулярных правил повысило полноту с ... до ...`
- `Доля полностью обезличенных запросов составила ...`

## Пример
```text
Исходный запрос:
Переведи 15 000 рублей с карты 1234 5678 9012 3456 на счёт Ивана Иванова

После обработки:
Переведи 15 000 рублей с карты [CARD_ID] на счёт [PERSON]
```

## Ограничения и развитие
- Качество зависит от покрытия словаря и правил;
- для редких форматов персональных данных нужны дополнительные шаблоны;
- следующий шаг — вынести оценку качества в отдельный воспроизводимый сценарий.
